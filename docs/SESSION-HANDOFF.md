- **Multipart real-boundary testing:** the `undici` package is **NOT importable** (Node bundles it internally but it's not a dependency) and the client base URLs are **hardcoded** in `src/config.ts` (not env-overridable), so the planned `undici` `MockAgent` approach doesn't work. Instead, the real-boundary test runs the client's exact `fetch` `init` through the global `new Request(url, init)` encoder (the same undici machinery `fetch` uses) and asserts the outbound `Content-Type` starts with `multipart/form-data; boundary=`. - **`file_path` upload is a deliberate server-side-read** (plan Decision 2) — safe ONLY under the STDIO trusted-caller model (the MCP client already holds the API key and is co-located). The tool descriptions document this. If this server is ever exposed over a non-STDIO/multi-tenant transport, `file_path` becomes an arbitrary-file-read primitive and must be gated or removed.
