## Known gotchas (carried)

- **MERGE-GATE GOTCHA — HIT AGAIN this session (5th time: #48, #50-s1, #50-s2, #69, now #67).** `gh pr merge <n> --squash --delete-branch` exited non-zero: it failed to delete the LOCAL branch ("used by worktree" — the **worktree**, not the shell CWD, holds it) AND skipped the remote-branch delete — **but the GitHub squash merge itself SUCCEEDED.** Don't trust the non-zero exit; verify with `gh pr view <n> --json state,mergedAt`. Recover manually from the MAIN repo root: `git worktree remove <path> --force` → `git branch -D <branch>` → `git worktree prune` → `git push origin --delete <branch>` → `rm -r .claude/menehune/locks/N.lock` → `git checkout main && git pull --ff-only origin main`. Verify: `git worktree list` (gone) + `git ls-remote --heads origin <branch>` (empty).
- **`/backlog:implement` ships the WHOLE issue as ONE PR.** #67's plan recommended a 4-slice PR split, but the implement command creates a single PR closing the issue — all 5 units landed together. If a multi-PR slice strategy is ever genuinely needed, drive the slices manually rather than via one `/backlog:implement` invocation.
- **Multipart real-boundary testing:** the `undici` package is NOT importable and the client base URLs are hardcoded in `src/config.ts`, so the `undici` `MockAgent` approach doesn't work. Run the client's exact `fetch` `init` through `new Request(url, init)` and assert the outbound `Content-Type` starts with `multipart/form-data; boundary=`.
- **`file_path` upload is a deliberate server-side-read** (#69) — safe ONLY under the STDIO trusted-caller model. If ever exposed over a non-STDIO/multi-tenant transport, `file_path` becomes an arbitrary-file-read primitive and must be gated or removed.
- **Spec-vs-plan shape drift:** roadmap/plan prose contains endpoint/shape guesses. Always re-verify against `docs/api/openapi-v2.yaml` at plan + implement time.
- **Auth reality (supersedes CLAUDE.md prose):** v2 uses the **`x-api-token` HEADER** (`src/client.ts`), NOT the `api_token` query param — that's v1-only. The shared `applyAuth` helper is the single source of truth. Integration `not.toContain('api_token')` assertions on v2 URLs are meaningful.
- **Lint IS a CI gate** (`.github/workflows/ci.yml` runs `npm run lint` = `eslint src/` — test files NOT linted in CI). Verify the real gate with `npx eslint src/`.
- **Adding a param means editing THREE places:** the Zod schema, the hand-written `inputSchema` JSON literal (NOT generated from Zod), and the handler's query/body/path forward line.
- **Zod 4.4.3 on main** (NOT Zod 3 — stale plan bodies may say "Zod 3.25"). Idioms: `z.email()`/`z.uuid()`, `z.record(z.string(), z.unknown())`, `.extend(...shape)` (not `.merge`), `error.issues` (not `.errors`).
- **`Buffer` → `Blob` typing:** wrap in a fresh `Uint8Array` (`new Blob([new Uint8Array(buffer)], {type})`) — runtime-correct and type-clean.
- **False-green hazard:** `tsconfig` excludes `**/*.test.ts` (tests never type-checked) and per-entity tests call handlers DIRECTLY (bypass the dispatcher's Zod). Real guards = UNIT schema tests (`Schema.parse`) + integration tests asserting the wire shape (`JSON.parse(options.body)` / the URL / `FormData` body) + the `tests/contract/` OpenAPI harness. (Note: handlers called directly skip Zod, so the schema's `.default(limit=50)` doesn't apply — the `buildPaginationParamsV2` default still yields `limit=50`.)
- **Unit schema tests live in `tests/unit/schemas/`** (3-level import `../../../src/...`), integration handler tests in `tests/integration/tools/`. Test helpers (`mockFetch.ts`, `mockEnv.ts`) live at **`tests/helpers/`** (NOT `tests/integration/helpers/`). `mockApiSuccess` returns the `vi.fn` mock so `mockFn.mock.calls[0]` exposes `[url, init]`. **`setupValidEnv()` sets `PIPEDRIVE_ENABLE_DESTRUCTIVE='true'` in `beforeEach`** — destructive-handler "enabled" tests work by default; "guard blocks" tests must `delete process.env.PIPEDRIVE_ENABLE_DESTRUCTIVE` explicitly.
